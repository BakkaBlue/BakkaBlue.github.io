## 🎮 游戏简介

**Idle Realms（放置王国）** 是一款浏览器端放置类 RPG 游戏。你扮演一位冒险者，通过伐木、采矿、钓鱼、烹饪、战斗等技能在奇幻世界中成长。游戏的核心特征是 **挂机累积进度** 和 **离线收益** —— 即使关闭浏览器，你的角色依然在努力修炼。

### 设计理念

- **低操作门槛**：选择行动后角色自动执行，无需持续操作
- **渐进式解锁**：技能等级提升解锁更高级的资源、怪物、地块
- **多维度成长**：10 种技能、装备系统、城镇经营、屠杀者任务、农务系统共同构成成长路径
- **暗黑奇幻美学**：暗金色主调、暗黑背景、3 种主题配色可选（暗金/翠绿/暗紫）
- **国际化支持**：中文 + English 双语言，数据-文本分离架构
- **右键交互**：背包/装备栏/商店全面支持右键菜单

## ⚔️ 核心玩法

### 游戏循环

```
选择动作 → 自动执行 → 获取资源/经验 → 升级技能 → 解锁新内容 → 更强装备 → 挑战更强怪物
```

1. 在左侧边栏选择一项技能（如"伐木"）
2. 在技能面板中选择一个动作（如"砍普通树木"）
3. 角色自动重复执行动作，每次完成后获得资源和经验
4. 等级提升后解锁更高级的动作（如 Lv.5 解锁"橡树"）
5. 将采集的资源烹饪成食物，或在商店出售换取金币
6. 购买/掉落装备武装自己，挑战更强怪物

### 经济循环

```
采集技能 → 原材料 → 加工技能 → 成品 → 金币/装备/食物
                                        ↓
                                  战斗/消耗/升级
```

## 🛠️ 技能系统

游戏包含 **10 种技能**，分为四大类别：

### 采集技能（Gathering）

| 技能 | 图标 | 描述 |
|------|------|------|
| **伐木** Woodcutting | 🌳 | 砍伐树木获得木材（原木 → 橡木 → 柳木 → 枫木） |
| **采矿** Mining | 💎 | 开采矿石（铜/锡 → 铁 → 煤 → 银 → 金） |
| **钓鱼** Fishing | 🐟 | 捕捞海产（虾 → 鲱鱼 → 鲈鱼 → 鲑鱼） |

### 加工技能（Production）

| 技能 | 图标 | 描述 |
|------|------|------|
| **烹饪** Cooking | 🍲 | 消耗生食制作熟食，恢复 HP |
| **锻造** Smithing | 🔨 | 需要采矿 Lv.10 解锁，v1 后续扩展 |
| **工艺** Crafting | ✂️ | 需要战斗 Lv.10 解锁，v2 扩展 |

### 领地经营

| 技能 | 图标 | 描述 |
|------|------|------|
| **农业** Farming | 🌱 | 购买地块、种植作物、升级地块加速生长 |
| **城镇** Town | 🏛️ | 建造/升级建筑获得全局加成，升级设施获取城镇资源 |

### 冒险

| 技能 | 图标 | 描述 |
|------|------|------|
| **战斗** Combat | ⚔️ | 挑战怪物获得金币、经验、装备掉落 |
| **屠杀者** Slayer | 💀 | 接取悬赏任务获得专属奖励和点数 |

### 经验公式

每个技能升级所需经验遵循公式 **xpForLevel(n) = 80 × n¹·⁶**：

| 等级 | 所需经验 | 累计经验 |
|------|----------|----------|
| 1→2 | 80 | 80 |
| 2→3 | 239 | 319 |
| 5→6 | 767 | ~1,538 |
| 10→11 | 1,812 | ~5,735 |

## ⚔️ 战斗系统

### 回合制战斗

- 每回合间隔 **1200ms**，玩家与怪物交替攻击
- 伤害公式：`伤害 = max(1, 攻击力 - 对方防御力 + rand(0~3) - 1)`
- 支持暴击（基础 5% 概率，1.5 倍伤害）
- 基础命中率 90%

### 怪物体系

| 怪物 | 战斗等级 | HP | 攻击 | 防御 | 特色掉落 |
|------|----------|-----|------|------|----------|
| 野生鸡 | 1 | 4 | 1 | 0 | 鸡毛、青铜剑(5%) |
| 野牛 | 3 | 10 | 2 | 1 | 牛皮 |
| 哥布林 | 8 | 22 | 5 | 2 | 哥布林之耳、青铜剑(5%) |
| 强盗 | 15 | 45 | 9 | 5 | 铁剑(5%)、铁胸甲(4%) |
| 骷髅战士 | 25 | 90 | 16 | 10 | 骨头、钢剑(4%)、钢胸甲(3%) |
| 巨魔 | 40 | 200 | 30 | 18 | 秘银剑(3%)、红宝石(2%)、力量项链(2%) |

### 战斗机制

- **自动战斗**：开启后自动连续挑战同一怪物
- **自动进食**：HP 低于 50% 时自动消耗食物恢复
- **逃跑**：可随时退出战斗
- **失败处理**：HP 归零后自动停止战斗，恢复到 30% HP
- **回血**：非战斗状态每 tick（100ms）有 5% 概率恢复 1 HP

## ⏱️ 离线收益

游戏关闭后，重新上线时会一次性结算离线期间的收益：

- **技能动作**：按动作耗时计算可完成的循环次数，自动结算材料和经验
- **烹饪**：受材料库存限制，材料耗尽自动停止
- **战斗**：根据玩家属性估算胜率，最多模拟 50 场，自动结算战利品
- **免费上限**：最多结算 **8 小时** 离线收益
- **保留进度**：完成整数次循环后保留余数进度，上线无缝衔接

## 🏗️ 项目架构

本项目采用 **数据驱动 + 事件总线** 的架构模式，核心逻辑与 UI 层完全分离：

```
┌─────────────────────────────────────────────┐
│                  UI Layer (Vue)              │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  │
│  │  Layout  │  │  Views   │  │Components │  │
│  │ TopBar   │  │SkillView │  │ StatChip  │  │
│  │ Sidebar  │  │CombatView│  │ProgressBar│  │
│  │SaveMenu  │  │ TownView │  │ ItemCard  │  │
│  │OfflineMod│  │ShopView  │  │  Toast    │  │
│  └──────────┘  └──────────┘  └───────────┘  │
├─────────────────────────────────────────────┤
│              Event Bus (松耦合通信)           │
├─────────────────────────────────────────────┤
│               Core Engine Layer              │
│  ┌──────────┐  ┌──────────────────────────┐ │
│  │GameEngine│  │        Systems            │ │
│  │OfflineEng│  │ Experience │ Combat      │ │
│  │SaveSystem│  │ Skill      │ Economy     │ │
│  │ EventBus │  │ Inventory  │ Equipment   │ │
│  │          │  │ Food       │ Town        │ │
│  │          │  │ Farming    │ Slayer      │ │
│  └──────────┘  └──────────────────────────┘ │
├─────────────────────────────────────────────┤
│              Data Layer (配置数据)            │
│  skills.ts │ items.ts │ monsters.ts         │
│  buildings.ts │ farming.ts │ slayer.ts     │
│  formulas.ts │ shop.ts │ settings.ts       │
├─────────────────────────────────────────────┤
│          State Layer (响应式状态)             │
│  GameState.ts │ SettingsState.ts │ types.ts │
└─────────────────────────────────────────────┘
```

### 关键设计决策

| 决策 | 说明 |
|------|------|
| **Vue reactive 作为全局状态** | `gameState` 是 `reactive<GameStateData>`，组件直接读取，核心层也可导入 |
| **EventBus 解耦** | UI 与 Core 通过事件通信：`item:acquired`、`combat:victory`、`skill:levelup` 等 |
| **纯数据配置** | 所有技能、物品、怪物均为 TypeScript 常量，新增内容只需添加配置 |
| **数据-文本分离** | game data 只存 ID + 数值，显示名由 i18n 层提供 |
| **100ms 游戏循环** | `GameEngine` 以 100ms 间隔驱动所有子系统 tick |
| **右键菜单** | 统一 ContextMenu 组件，背包/装备栏/商店均支持右键交互 |
| **无运行时依赖** | 仅依赖 Vue 3 + vue-i18n，无后端、无数据库，纯前端项目 |

## 🌐 国际化 (i18n)

本项目已全面支持多语言，使用 **vue-i18n**（Composition API 模式）。

### 语言支持

| 语言 | 覆盖度 | 状态 |
|------|--------|------|
| 🇨🇳 中文 (zh-CN) | 100% | ✅ 全文翻译 |
| 🇺🇸 English (en-US) | ~20% | ⚠️ 基础词条已翻译，其余待补 |

### 架构

```
src/i18n/
├── index.ts              # createI18n 初始化
├── types.ts              # MessageSchema 类型（以 zh-CN 为 source of truth）
├── locales/
│   ├── zh-CN/            # 12 个模块文件，~350+ key
│   │   ├── common.ts     # 通用词条（金币/等级/保存/取消...）
│   │   ├── items.ts      # 45 个物品名
│   │   ├── skills.ts     # 10 技能 + 18 动作名
│   │   ├── monsters.ts   # 6 个怪物名
│   │   ├── buildings.ts  # 4 建筑（名+描述+效果模板）
│   │   ├── facilities.ts # 2 设施（名+描述+资源名）
│   │   ├── town-shop.ts  # 4 商品 + 标签
│   │   ├── farming.ts    # 3 地块类型 + 8 作物名
│   │   ├── slayer.ts     # 3 导师 + 5 奖励 + 7 特殊掉落
│   │   ├── ui.ts         # 全部 UI 文本（布局/战斗/装备/背包/农务/城镇/设置/日志）
│   │   ├── toasts.ts     # 系统 Toast 消息模板
│   │   ├── settings.ts   # 默认玩家名
│   │   └── index.ts      # 汇总导出
│   └── en-US/
│       └── index.ts      # 英文桩（渐进式翻译）
```

### 数据-文本分离

所有游戏数据（`data/` 层）只存 `id` 和数值，显示名通过 `t(\`items.${id}\`)` 动态获取。

```typescript
// 数据层 — 只存 ID + 数值
{ id: 'mithril_sword', value: 2500, atk: 28, rarity: 'rare' }

// UI 层 — 自动显示对应语言
t(`items.${id}`)  // 中文: '秘银剑'  / English: 'Mithril Sword'
```

### 添加新内容只需

```
1. data/xxx.ts → { id: 'new_thing', ...数值 }
2. i18n/zh-CN/xxx.ts → new_thing: '新东西'
3. i18n/en-US/index.ts → items: { new_thing: 'New Thing' }
```

UI 代码零改动。

### 语言切换

设置中心 → 个性化 → 语言 / Language 下拉选择，即时生效，刷新后 localStorage 自动恢复。

## 🖱️ 右键菜单系统

### ContextMenu 组件

通用右键浮动菜单，支持：
- **边界检测**：菜单超出屏幕右侧/底部时自动偏移
- **分割线**：`divider: true` 渲染分隔符
- **入场动画**：`scale(0.95)→1` + opacity 过渡
- **ESC 关闭**：键盘 Esc 或点击菜单外任意位置关闭

### 背包右键

| 物品类型 | 左键 | 右键菜单 |
|:---|:---|:---|
| **装备** (武器/防具/饰品) | 切换装备状态 | [检查] [装备/卸下] [出售] — [丢弃] |
| **食物** (heal>0) | 食用 | [检查] [食用] [出售] — [丢弃] |
| **材料/杂项** | 查看详情 | [检查] [出售] — [丢弃] |

### ItemDetail 物品详情弹窗

右键"检查"打开居中弹窗，展示：
- 大图标（稀有度颜色边框）
- 属性行：攻击力/防御力/恢复/价值/槽位
- 操作按钮：装备/使用/出售/关闭

### 商店右键

左键购买，右键查看物品详情弹窗。

### 装备栏右键

- **已装备物品**：右键 → [卸下] [替换...]
- **空槽位**：右键 → 弹出同槽位装备列表 + **搜索框**

搜索框实时过滤物品名称，候选装备按属性从高到低自动排序。

## ⚙️ 设置中心

完整的游戏设置面板（侧栏 → 设置），4 个标签页：

### 通用设置
- ☑ **自动保存**（开关）
- **自动保存频率**（30s / 60s / 2min / 5min）
- ☑ **显示右侧信息栏**（控制右侧当前活动和日志面板显隐）
- ☑ **减少动态效果**（全局禁用 CSS 动画/过渡，适合低端设备）

### 个性化
- **界面主题**：暗金 / 翠绿 / 暗紫（三选一，即时切换 CSS 变量）
- **字体大小**：小(12px) / 中(14px) / 大(16px)
- **语言 / Language**：🇨🇳 中文 / 🇺🇸 English

### 玩家设置
- **角色名称**（可编辑，最长 12 字）
- **导出存档码**（一键复制到剪贴板）
- **导入存档码**（跨设备转移数据）

### 存档管理
- 3 个存档槽位：显示名称、总等级、最后保存时间
- 复制存档码 / 删除存档
- ⚠️ **一键核平**（清除全部数据，双重确认）

## 📝 设计说明

### 数据驱动设计

所有游戏内容（技能动作、物品装备、怪物、建筑、作物）均为 TypeScript 常量配置。添加新内容只需在对应 `src/data/*.ts` 文件中追加条目，无需修改任何逻辑代码。

例如，添加一种新怪物只需在 `monsters.ts` 中新增一个 `MonsterConfig` 对象——战斗系统、离线引擎、屠杀者任务系统会自动识别。

### 离线收益计算

离线引擎 `OfflineEngine.ts` 在玩家上线时运行，计算离线期间的收益：

1. 读取 `lastSaveTime`，计算离线时长
2. 根据当前正在执行的动作类型（技能/战斗），模拟离线期间的完成次数
3. 技能动作受材料库存限制（如烹饪耗尽生食则停止）
4. 战斗使用简化模型估算胜率和回合数，分批结算
5. 保留进度余数，上线后从剩余时间继续

### 存档系统

- **3 个存档槽位**，存储在 `localStorage`
- **索引分离**：`idleRealms_index` 存元信息（名称/等级/时间），各槽位独立存储
- 使用 **Base64 + encodeURIComponent** 编码防轻易篡改
- 自动保存间隔 60 秒（可配置）
- 支持 `patchMissingFields` 向前兼容（旧存档加载时自动补全新字段）
- 支持 **导入/导出** 存档数据（设置中心 → 玩家 → 数据管理）
- **存档与 i18n 解耦**：存档只存 item ID，读取时通过 `t()` 实时获取当前语言的名称

### 主题系统

支持三种主题，通过 CSS 变量实现：

| 主题 | 主色调 | 风格 |
|------|--------|------|
| **默认（暗金）** | `#e6b566` 暗金色 | 暗黑奇幻 |
| **森林** | `#66bb6a` 翠绿色 | 自然清新 |
| **魔法** | `#b968e0` 紫罗兰 | 神秘魔法 |
